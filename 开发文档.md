💡 核心创新点：OpenMonkey 的“万能技能适配器” (Universal Skill Adapter)

目前的痛点是：
格式割裂：Claude Skills 是一套格式，OpenClaw 是一套，Cursor Rules 又是一套，三万 (Sanwan) 又有自己的指令集。
创建门槛高：用户想加个技能，得懂 Prompt Engineering，得会写 Markdown 结构。
无法互通：你在 Claude 上配好的技能，没法直接用到本地助手上。

OpenMonkey 的解决方案：
内置“技能爬虫/解析器”：能读取 Sanwan.ai、GitHub (OpenClaw Skills)、Anthropic Skills 等主流库的技能描述。
LLM 驱动的“转译引擎”：
    输入：任意格式的技能描述（网页链接、Markdown、甚至一段自然语言对话）。
    处理：OpenMonkey 调用本地 LLM，分析该技能的意图、触发条件、执行步骤、约束规则。
    输出：标准化为 OpenMonkey 原生技能格式 (OM-SKILL.md)，并自动存入 skills/ 目录。
自适应学习：如果导入的技能在本地运行失败（例如依赖了本地没有的工具），OpenMonkey 会自动反思，修改技能文件，尝试用本地已有的工具（如 Rust 内置命令、本地脚本）替代，直到技能可用为止。

🛠️ 技术栈 (Strict & Minimal)
组件   选型   理由
Core   Rust 1.80+   性能、安全、跨平台。

Desktop   Tauri v2   单二进制，原生体验。

Frontend   React + TS + Tailwind   快速迭代，生态丰富。

State   Zustand   轻量级状态管理。

Editor   Monaco Editor   专业代码/Markdown 编辑体验。

DB   SQLx + SQLite   仅用于记忆存储 (memories 表)，支持 sqlite-vec (可选)。

HTTP   Reqwest   异步请求，支持 SSE。

Security   Keyring   系统级密钥环存储 API Key。

File Watch   Notify   监听文件变化，实现热重载。

❌ 严禁使用: WASM 沙箱 (v2.0 暂不需要), 重型向量库 (Chroma/Milvus), 复杂消息队列。
